### Key Findings
1. **GPU Contention:**
   - Single RTX 4090 (24GB): PersonaPlex + Ollama 7B fits; time-slicing adds ~100-200ms latency
   - Dual GPU (RTX 4090 × 2): Ideal setup (GPU 0 = PersonaPlex, GPU 1 = Ollama)
   - CPU fallback: Unacceptable for 7B models (~2-5 seconds per inference)

2. **VRAM Budget:**
   - PersonaPlex (Mimi encoder/decoder + 7B LM): ~12-14GB
   - Ollama llama3.3-8B: ~8-10GB
   - Total: ~20-24GB (fits single RTX 4090 with time-slicing)

3. **Agent Framework Recommendation:**
   - **Winner:** Microsoft.Extensions.AI (M.E.AI)
   - Rationale: Native .NET 9+, lighter-weight, chat completion + tool calling support
   - Semantic Kernel: More mature plugin ecosystem but heavier
   - AutoGen: Python-first (not ideal for .NET stack)

4. **Latency Breakdown (Mode B with llama3.3-8B):**
   - Audio capture: ~100ms
   - Mimi encoder: ~50ms
   - PersonaPlex 7B LM: ~200-400ms
   - M.E.AI orchestration: ~100ms
   - Ollama 8B: ~200-400ms (vs 500-1500ms for 70B)
   - Mimi decoder: ~50ms
   - Network + playback: ~70ms
   - **Total:** ~770-1170ms (acceptable for "thinking" mode)

5. **PersonaPlex Pipeline Gaps:**
   - ONNX export (7B LM) in progress — blocks full pipeline
   - File-based API prevents streaming — requires memory refactor
   - No exposed streaming tokens — full-duplex requires architecture change
   - Moshi audio codec (MP3-based) — efficient but adds codec latency

### Conversation Flow Patterns
- **Mode A (Direct):** User speech → PersonaPlex → AI speech (~490-690ms)
- **Mode B (Augmented):** User speech → PersonaPlex ASR → Agent Framework → Ollama → PersonaPlex TTS → AI speech (~770-1170ms with 8B, 1090-2390ms with 70B)
- Recommendation: Start Mode A for MVP; Mode B as opt-in "deep thinking"

### Mitigation Strategies
1. Use dual-GPU setup or implement GPU queueing service
2. Select smaller Ollama models (8B vs 70B) to stay under latency budget
3. Implement streaming responses (LLM token streaming → early TTS start)
4. Use visual feedback ("thinking...") to mask latency
5. Time PersonaPlex calls before Ollama (warm-up GPU before switching)

### Technology Decisions
1. **Agent Framework:** Microsoft.Extensions.AI (M.E.AI) over Semantic Kernel
2. **Ollama Target:** llama3.3-8B for Phase 1 (upgrade to 70B for Phase 3 if GPU allows)
3. **GPU Setup:** Dual-GPU recommended; single GPU acceptable with queueing
4. **Streaming:** Implement in Phase 2 (requires PersonaPlex API refactor)
